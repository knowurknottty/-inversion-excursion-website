# RCR PROTOCOL v2.0
## Review-Critique-Revise: Refined Process

**Created:** 2026-03-06  
**Purpose:** Maximum depth analysis within context constraints  
**Model:** Reasoning at maximum

---

## PRINCIPLES

1. **Time Integrity** - Track actual time, not aspirational time
2. **Depth over Breadth** - One element fully analyzed beats five elements shallowly reviewed
3. **Evidence-Based** - Every critique must cite specific text
4. **Context Awareness** - Monitor token usage; pause before limits
5. **Honest Reporting** - If time runs out mid-analysis, document exactly where we are

---

## PHASE 1: REVIEW (25% of time)

### Step 1.1: Full Read-Through
- Read the entire target section/document
- NO editing, NO analysis - just intake
- Note initial emotional reactions (boredom, confusion, excitement)
- Note obvious gaps ("this section ends abruptly")

### Step 1.2: Structural Mapping
- Map the architecture: sections, subsections, flow
- Identify the promise: what does this section claim to deliver?
- Check for delivery: does it fulfill that promise?
- Note word count per section (approximate)

### Step 1.3: Surface Inventory
- List all major elements (tables, lists, examples, etc.)
- Check formatting consistency
- Note any obvious errors (typos, broken references)

**Deliverable:** Review Notes document with:
- Promise vs Delivery assessment
- Structural map
- Surface issues list
- Time check: actual minutes elapsed

---

## PHASE 2: CRITIQUE (50% of time)

### Step 2.1: The Deep Questions
For EACH major element, ask:

**Content Questions:**
- Is this true? (if factual claim)
- Is this clear? (if explanation)
- Is this necessary? (if section)
- Is this sufficient? (if list/example)

**Impact Questions:**
- Does this engage the reader?
- Does this advance the argument?
- Does this provoke thought or just inform?
- Would a reader remember this tomorrow?

**Integration Questions:**
- Does this connect to other chapters?
- Does this reference established concepts?
- Does this set up future sections?

### Step 2.2: Gap Analysis
- What's missing that should be here?
- What's here that shouldn't be?
- What questions would a reader have that aren't answered?
- What would make this unforgettable?

### Step 2.3: Prioritize Critiques
Rank findings by severity:
- **CRITICAL:** Must fix (false claims, major gaps, broken promises)
- **IMPORTANT:** Should fix (weak examples, unclear explanations)
- **NICE:** Could improve (stylistic, additional depth)

**Deliverable:** Critique Report with:
- Cited passages for each critique
- Severity rankings
- Specific recommendations
- Time check: actual minutes elapsed

---

## PHASE 3: REVISE (25% of time)

### Step 3.1: Critical Fixes First
- Address all CRITICAL items
- If time runs out, document which CRITICAL items remain

### Step 3.2: Important Improvements
- Address IMPORTANT items if time allows
- Focus on highest-impact changes

### Step 3.3: Integration Check
- Ensure new material flows with existing
- Check cross-references
- Verify formatting consistency

**Deliverable:** Revised document + Revision Log:
- List of changes made
- List of changes deferred (with reason)
- Time check: actual minutes elapsed

---

## TIME MANAGEMENT RULES

### Strict Timeboxing
- Set timer at start of each phase
- When timer expires, stop current phase
- Document exactly where you are
- Move to next phase or end session honestly

### Token Monitoring
- Check context usage every 5 minutes
- At 70% context, begin winding down
- At 80% context, complete current task only
- At 90% context, stop and summarize

### Honest Reporting
**Required end-of-session report:**
```
TIME ACTUAL: X minutes
PHASES COMPLETED: Review [Y/N], Critique [Y/N], Revise [Y/N]
CRITICAL ITEMS: Fixed [N], Remaining [N - list them]
IMPORTANT ITEMS: Addressed [N], Remaining [N]
CONTEXT USED: X%
NEXT SESSION PRIORITIES: [list]
```

## POST-SESSION WORKFLOW

1. Update RCR document with actual times
2. Commit changes with honest message
3. Create summary of what was/wasn't completed
4. Set priorities for next session if incomplete

---

## INTEGRITY CHECKLIST

Before reporting completion, verify:
- [ ] Time claimed matches actual elapsed time
- [ ] All critical issues addressed OR explicitly listed as remaining
- [ ] Context usage reported honestly
- [ ] No material claims about work not done

**Violation:** Claiming "30 minutes RCR complete" when actual time was 15 minutes.
**Correction:** Report "15 minutes RCR - Phase 1 and 2 complete, Phase 3 partial"

---

*The quality of the work matters more than the speed of completion.*
*Honest reporting builds trust. Inflated claims destroy it.*
