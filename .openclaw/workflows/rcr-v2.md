# CUSTOM WORKFLOW: RCR PROTOCOL v2.0
## Review-Critique-Revise System

**Created:** 2026-03-06  
**Purpose:** Maximum depth analysis within context constraints  
**For Use With:** Book chapters, dungeons, critical documents

---

## QUICK START

```
/time budget [minutes]
/load rcr-v2
/begin [document name]
```

---

## PHASE 1: REVIEW (25% of time)

### 1.1 Full Read-Through
- [ ] Read entire document without editing
- [ ] Note emotional reactions (boredom, confusion, excitement)
- [ ] Note obvious gaps

### 1.2 Structural Mapping
- [ ] Map sections and subsections
- [ ] Identify the promise
- [ ] Check promise vs delivery
- [ ] Approximate word count per section

### 1.3 Surface Inventory
- [ ] List all major elements (tables, lists, examples)
- [ ] Check formatting consistency
- [ ] Note obvious errors

**Deliverable:** Review Notes with structural map

---

## PHASE 2: CRITIQUE (50% of time)

### 2.1 Deep Questions (per element)
**Content:**
- Is this true?
- Is this clear?
- Is this necessary?
- Is this sufficient?

**Impact:**
- Does this engage?
- Does this advance?
- Does this provoke?
- Will this be remembered?

**Integration:**
- Does this connect to other chapters?
- Does this reference established concepts?
- Does this set up future sections?

### 2.2 Gap Analysis
- [ ] What's missing?
- [ ] What's unnecessary?
- [ ] What questions aren't answered?
- [ ] What would make this unforgettable?

### 2.3 Prioritize
- **CRITICAL:** Must fix (false claims, broken promises)
- **IMPORTANT:** Should fix (weak examples, unclear)
- **NICE:** Could improve (stylistic)

**Deliverable:** Critique Report with cited passages

---

## PHASE 3: REVISE (25% of time)

### 3.1 Critical Fixes
- [ ] Address all CRITICAL items
- [ ] If time runs out, document remaining

### 3.2 Important Improvements
- [ ] Address IMPORTANT if time allows
- [ ] Focus on highest impact

### 3.3 Integration Check
- [ ] Ensure new material flows
- [ ] Check cross-references
- [ ] Verify formatting

**Deliverable:** Revised document + Revision Log

---

## TIME RULES

### Strict Timeboxing
- Set timer at start
- When timer expires, stop
- Document exactly where you are
- Report honestly

### Token Monitoring
- Check context every 5 minutes
- At 70%: begin winding down
- At 80%: complete current task only
- At 90%: stop and summarize

### Honest Reporting (Required)
```
TIME ACTUAL: X minutes
PHASES COMPLETED: Review [Y/N], Critique [Y/N], Revise [Y/N]
CRITICAL ITEMS: Fixed [N], Remaining [N - list]
CONTEXT USED: X%
NEXT SESSION: [priorities]
```

## DUNGEON-SPECIFIC CHECKLIST

**Architecture:**
- [ ] Entrance transition from previous dungeon
- [ ] Boss introduction with threat escalation
- [ ] Mudrā integration clear and actionable
- [ ] Dialogue table with effective responses
- [ ] Defeat scene/resolution moment
- [ ] Exit transition to next dungeon

**Content:**
- [ ] No forward references to future chapters
- [ ] Concrete examples (not just abstract claims)
- [ ] Safety warnings where needed
- [ ] Speculative claims properly tagged
- [ ] Frequencies match SynSync protocols

---

## INTEGRITY CHECKLIST

Before reporting completion:
- [ ] Time claimed matches actual elapsed
- [ ] All critical issues addressed OR listed
- [ ] Context usage reported
- [ ] No claims about undone work

**Violation Example:** "30 minutes RCR complete" when actual was 15 minutes.  
**Correction:** "15 minutes RCR - Phase 1-2 complete, Phase 3 partial"
