```
/dungeon/pharisee-temple                   [Dungeon 4]
/dungeon/bio-laboratory                    [Dungeon 5]
/dungeon/ice-fortress                      [Dungeon 6]
/dungeon/qanon-labyrinth                   [Dungeon 7]
```

**Option B: Hybrid Structure (EASIER MIGRATION)**
```
/chapter/the-ivory-tower                   [Chapter 1]
/chapter/five-scrolls-elemental-keys       [Chapter 2]
/chapter/seven-dungeons-guide              [Chapter 3]
/chapter/master-keys-advanced              [Chapter 4]
/chapter/ascension-practices               [Chapter 5]
/chapter/grimoire-wyrd                     [Chapter 6]
/chapter/transmission                      [Chapter 7]
```

### 2.3 URL Structure Justification

**Why Semantic Over Numeric:**
- Keywords in URLs are a ranking factor (minor but real)
- `/the-ivory-tower` is more memorable than `/chapters/chapter-1`
- Social sharing benefits from descriptive URLs
- Better click-through rates from SERPs

**Why Hybrid May Be Safer:**
- Easier to maintain with existing file structure
- Clear chapter hierarchy for users
- Simple redirects if content evolves

### 2.4 Implementation Priority
1. **Phase 1:** Create redirects from `/chapters/chapter-N.html` → new semantic URLs
2. **Phase 2:** Update all internal links
3. **Phase 3:** Submit updated sitemap to Google
4. **Phase 4:** Monitor for 404s and fix broken backlinks
